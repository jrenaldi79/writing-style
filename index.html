01 / Concepts

What Are Skills?

01

Specialized Capabilities

Skills are pre-built workflows that extend your AI's abilities. Instead of explaining complex tasks each time, skills encode domain knowledge and step-by-step processes.

02

Progressive Disclosure

Skills load information in layers - only what's needed, when it's needed. This keeps context clean and responses fast, even for complex multi-step workflows.

03

Local Automation

Skills can run Python scripts on your machine, access local files, and integrate with tools like Gmail - all without consuming AI tokens for every operation.

04

State Persistence

Skills remember where you left off. Start a task today, continue tomorrow. Progress is saved locally in state.json.

06 / Efficiency

Progressive Disclosure

Skills use a 3-level loading strategy to minimize context usage.

Level What Loads When
1. Metadata Skill name, triggers, description Always (in system prompt)
2. Workflow SKILL.md with full instructions When skill is triggered
3. References Supporting docs, schemas, examples Only when specifically needed
Result: 80%+ reduction in context usage compared to loading everything upfront.

08 / Support

Troubleshooting

"Skill not found"
Check that the skill is installed in ~/skills/{skill-name}/ and added to your system prompt
Skill doesn't trigger
Verify the trigger phrase is in your system prompt's Available Skills table
Lost progress
Check state.json in the skill's data directory - say "Show project status"
Script errors
Make sure Python venv is set up in the data directory
